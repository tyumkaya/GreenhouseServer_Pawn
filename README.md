## Описание

В репозитории находится серверная реализация системы теплиц для open.mp / Pawn.

Реализовано:

- теплица работает только пока владелец находится на сервере
- загрузка теплиц после успешного логина игрока
- выгрузка и сохранение при дисконнекте
- ограничение: не более 5 теплиц на одного игрока
- взаимодействие с теплицей только рядом с ней
- полное созревание урожая за 10 минут
- улучшение теплицы с ускорением роста x2
- визуализация стадий роста возле теплицы
- сохранение теплиц в базе данных

## Что входит в репозиторий

Репозиторий содержит:

- исходный код сервера
- SQL-файлы для создания нужных таблиц
- документация по запуску и проверке

Репозиторий не содержит:

- живую базу данных
- реальные данные подключения к БД
- клиент GTA SA
- лаунчер open.mp
- пользовательские локальные файлы и логи
- компилятор pawncc

## Требования

Для запуска нужны:

- open.mp server
- Pawn compiler
- MySQL
- плагин MySQL
- плагин streamer
- плагин Pawn.CMD
- плагин Pawn.Regex
- плагин mdialog

## Структура проекта

Примерная структура:

```text
gamemodes/        - основной код gamemode
plugins/          - плагины сервера
components/       - дополнительные модули проекта
sql/              - SQL-схемы и миграции
docs/             - документация по проекту
scriptfiles/      - runtime-данные сервера
```

## Подготовка базы данных

1. Создайте базу данных.
2. Импортируйте SQL-файлы из папки `sql/`.
3. Убедитесь, что в базе есть таблицы:
   - `users`
   - `greenhouses`
  
## Настройка подключения к БД

- Отредактируйте файл ```server/settings.inc```, введя данные своей БД

## Сборка

Скомпилируйте основной скрипт ```gamemodes/gamemodes.pwn``` через pawncc или Qawno.

## Взаимодействие

Игрок может взаимодействовать с теплицей только рядом с ней.
Проверка близости выполняется на сервере.

Тестовые команды:

- /ghcreate — создать теплицу на позиции игрока
- /ghplant — создать теплицу на позиции игрока
- /ghlist — показать список теплиц игрока
- /ghstatus — информация о ближайшей теплице
- /ghforce — мгновенно получить последнюю стадию созревания
- /ghreset — сбросить созревание урожая теплицы
- /ghupgrade — улучшить теплицу
- /ghharvest — собрать урожай
### Важно!
Команды работают только при активной учётке RCON админа. 
Для того чтобы активировать учётку админа, надо прописать команду:
- ```/rcon login 123123``` (или тот пароль, который будет указан в ```config.json```)

## Как проверить

Рекомендуемый сценарий проверки:

1. Запустить сервер.
2. Подключиться к серверу.
3. Зарегистрироваться или войти в существующий аккаунт.
4. Создать теплицу стоя в открытой местности (```/ghcreate```)
5. Убедиться, что запись появилась в таблице greenhouses.
6. Посадить помидоры (```/ghplant```)
7. Проверить, что рядом с теплицей доступно взаимодействие.
8. Дождаться изменения стадии роста.
9. Проверить улучшение x2. (```/ghupgrade```)
10. Проверить, что при выходе игрока состояние сохраняется.
11. Зайти снова и убедиться, что теплица загружается корректно.
12. Проверить, что оффлайн-прогресс не происходит.
13. Проверить ограничение в 5 теплиц на одного игрока.
